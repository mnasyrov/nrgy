# Документация для разработчиков

## Назначение

Этот раздел содержит гайды по разработке, стандарты документации и инструкции
для агентов, используемые в репозитории Nrgy.js.

## Общая информация

Материалы из `docs/developers` задают правила структуры исходников,
документирования, форматирования и сопровождения кода. Они предназначены для:

- авторов пакетов внутри монорепозитория;
- контрибьюторов, изменяющих исходный код и тесты;
- автоматизаций и AI-агентов, генерирующих документацию или кодовые правки.

## Состав раздела

- [Требования к документации](./docs_requirements.ru.md): обязательные правила
  структуры и размещения документации модулей и пакетов.
- [Prompt для документации](./docs_prompt.ru.md): шаблон prompt'а для агентов,
  генерирующих документацию модулей.
- [Стиль кодирования](./coding_style.ru.md): практические соглашения, выведенные
  из текущего кодстайла проекта.
- [Процесс разработки](./development_workflow.ru.md): команды, проверки и
  рабочий процесс по репозиторию.
- [Гайд для агентов](./agent_guide.ru.md): ожидания от автоматизированных
  участников и documentation-агентов.

## Примечания по использованию

- При добавлении или изменении исходных файлов обновляйте colocated
  документацию в той же директории.
- Не создавайте отдельную документацию для `index.ts`; публичные точки входа
  описывайте в пакетных `README.md`.
- Держите английскую и русскую документацию синхронизированной.
